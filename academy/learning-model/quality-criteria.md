# Educational Quality Criteria

Academy modules must meet a consistent quality standard before publication.

## Technical Accuracy

- Concepts are technically correct
- Mathematical notation is consistent
- Code executes successfully
- Framework APIs are current
- Claims are supported
- Limitations are stated clearly

## Educational Clarity

- The intended learner level is clear
- Prerequisites are explicit
- Learning objectives are measurable
- Concepts are introduced progressively
- Unnecessary jargon is avoided
- Examples reinforce the learning objectives

## Practical Quality

- Setup instructions are complete
- Dependencies are reproducible
- Notebooks execute in the documented order
- Expected results are described
- Random or probabilistic results are interpreted correctly
- Exercises include validation criteria

## Assessment Alignment

- Every primary objective is assessed
- Assessment difficulty matches the module
- Exercises precede solutions
- Challenges require application rather than copying
- Passing criteria are explicit

## Source Quality

- Primary sources are preferred
- Official documentation is used for framework behavior
- Academic claims use suitable references
- External assets include attribution and licensing information
- Unsupported marketing claims are excluded

## Framework Neutrality

Where practical, modules should distinguish:

- Quantum computing concepts
- Framework-specific implementation
- Provider-specific behavior
- Hardware-specific limitations

A module should not present one vendor's implementation as the universal
definition of a quantum concept.

## Accessibility

Content should:

- Use clear headings
- Explain symbols and notation
- Avoid relying only on color
- Include text explanations for diagrams
- Use readable notebook output
- Avoid unnecessarily complex language

## Reproducibility

A published module must document:

- Required Python version
- Required dependencies
- Execution commands
- Random seeds where relevant
- Simulator or backend configuration
- Expected output or output range
- Known environment limitations
