# Standard Module Structure

Every Academy curriculum module must follow a consistent structure.

## Required Module Elements

### 1. Overview

The overview must state:

- Module title
- Curriculum level
- Current status
- Estimated duration
- Difficulty
- Prerequisites
- Required tools
- Supported frameworks

### 2. Learning Objectives

Each module must define measurable learning objectives describing what the
learner should be able to explain, calculate, implement or evaluate.

### 3. Prerequisite Check

The module must identify the knowledge and technical capabilities required
before starting.

A prerequisite check may include:

- Short questions
- Small Python exercises
- Mathematical checks
- References to earlier modules

### 4. Theory

Theory content must explain the relevant concepts before introducing
framework-specific implementation.

### 5. Mathematics

Mathematics content must include only the material required to achieve the
module objectives.

It should:

- Define notation
- Explain variables and assumptions
- Include worked examples
- Connect mathematical expressions to quantum concepts

### 6. Visual Explanation

Where appropriate, modules should include:

- Circuit diagrams
- State-vector diagrams
- Bloch sphere representations
- Probability distributions
- Execution flow diagrams
- Conceptual illustrations

### 7. Guided Implementation

The learner should build a complete implementation with explanations for each
important step.

### 8. Interactive Notebook

The notebook should allow learners to modify parameters, rerun experiments and
observe how results change.

### 9. Exercises

Exercises must reinforce the learning objectives and should increase
progressively in difficulty.

### 10. Challenge

Each module should include a practical challenge requiring the learner to apply
multiple concepts without a complete step-by-step solution.

### 11. Assessment

The assessment must verify that the learner has achieved the stated learning
objectives.

### 12. References

References should prioritize:

- Official framework documentation
- Academic textbooks
- Peer-reviewed papers
- Primary technical sources
- Recognized educational institutions

### 13. Next Steps

Each module should explain:

- What the learner has completed
- Which skills were acquired
- Which module should follow
- Which optional resources may deepen understanding

## Optional Module Elements

Depending on the subject, a module may include:

- Historical context
- Framework comparison
- Simulator comparison
- Hardware execution
- Noise analysis
- Performance benchmark
- Research extension
- Instructor notes
