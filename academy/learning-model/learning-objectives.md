# Learning Objectives

Learning objectives define the expected learner outcomes for a module.

## Objective Requirements

Each objective must be:

- Specific
- Observable
- Measurable
- Appropriate for the curriculum level
- Verifiable through exercises or assessment

## Recommended Verbs

### Understanding

- Define
- Describe
- Explain
- Identify
- Distinguish
- Summarize

### Application

- Calculate
- Construct
- Demonstrate
- Execute
- Implement
- Simulate
- Use

### Analysis

- Analyze
- Compare
- Diagnose
- Evaluate
- Interpret
- Measure
- Validate

### Creation and Research

- Design
- Develop
- Formulate
- Optimize
- Reproduce
- Investigate
- Propose

## Avoid Weak Objectives

Avoid objectives such as:

- Learn quantum gates
- Understand Qiskit
- Become familiar with entanglement

These cannot be assessed reliably.

Prefer:

- Explain how the Hadamard gate transforms the computational basis states
- Construct and simulate a two-qubit Bell-state circuit
- Compare ideal and noisy measurement distributions
- Interpret the output counts of a shot-based simulation

## Objective Count

A typical module should define between four and eight primary objectives.

Very large modules should be divided rather than given an excessive number of
objectives.

## Objective-to-Assessment Alignment

Every primary learning objective must be evaluated by at least one of:

- Exercise
- Practical task
- Quiz question
- Module challenge
- Final assessment

No primary objective should exist without a corresponding validation method.
