# Module Lifecycle

The module lifecycle defines how Academy modules are proposed, developed,
reviewed, published, maintained and retired.

## Status Values

### Proposed

The module idea exists but scope and ownership are not yet approved.

### Planned

The module is accepted into the roadmap and has a defined purpose.

### In Development

Content, notebooks, exercises or assessments are being created.

### Review

The module is undergoing technical, educational and editorial review.

### Pilot

The module is available for limited learner testing and feedback.

### Published

The module is approved for general use.

### Maintenance

The module remains published but requires dependency, API or content updates.

### Deprecated

The module is still accessible but no longer recommended.

### Archived

The module is retained for historical purposes and is no longer maintained.

## Required Review Areas

Before publication, a module should be reviewed for:

- Technical accuracy
- Educational clarity
- Mathematical correctness
- Code quality
- Reproducibility
- Framework compatibility
- Reference quality
- Accessibility
- Assessment alignment

## Change Categories

### Editorial Change

Examples:

- Typographical corrections
- Improved wording
- Formatting
- Clarified explanations

### Minor Content Change

Examples:

- New example
- Additional exercise
- Updated visualization
- Small API update

### Major Content Change

Examples:

- Changed learning objectives
- New prerequisite
- Replaced framework
- Reorganized module sequence
- Changed assessment standard

Major changes require a complete review of learning-objective and assessment
alignment.

## Retirement

A module may be deprecated or archived when:

- The technology is obsolete
- The content is technically incorrect
- The module is replaced
- Maintenance is no longer practical
- The learning objective no longer fits the curriculum
